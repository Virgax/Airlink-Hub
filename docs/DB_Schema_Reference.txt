## TABLE: Posiciones (DB: SPN)
================================================================
Key columns used in Hub/LX system:

| Column                | Notes                                    |
|-----------------------|------------------------------------------|
| Codigo                | PK — código de posición (FK en Empleados.Posicion) |
| descripcion           | Nombre de la posición                    |
| Estatus               | Activo/Inactivo                          |

Full column list:
Codigo, descripcion, Referencia, PermitirIngresoExtra, Ano, MontoEncuesta,
Formula_SalarioHora, Codigo_Alternativo, Ruta_Documento, UrlVideo, Estatus,
Intermitente, Formula_Salario_Diario, Posicion_Critica, Dias_Cobertura_Plaza

Join to Empleados:
  Empleados.Posicion = Posiciones.Codigo

## TABLE: Departamentos (DB: SPN)
================================================================
Key columns used in Hub/LX system:

| Column                | Notes                                    |
|-----------------------|------------------------------------------|
| ID                    | PK — FK en Empleados.ID_Departamento     |
| codigo                | Código del departamento                  |
| descripcion           | Nombre del departamento                  |
| Codigo_Encargado      | Código del jefe del depto                |

Full column list:
ID, codigo, descripcion, Codigo_Encargado, AgregarDiasSalarioAdicional,
Referencia, Departamento_Grupo, Facilidad, RRHH

Join to Empleados:
  Empleados.ID_Departamento = Departamentos.ID

## STORED PROCEDURES — DB: SPN
================================================================

SP_Hub_ValidateLogin
  @EmployeeCode varchar(20)
  @LastFour     varchar(4)
  Returns: IsValid, EmployeeCode, EmployeeName, EmployeeEmail,
           PositionName, DepartmentName, ReportLevel
  Security: Returns empty strings for all fields if LastFour doesn't match

================================================================
## DB: AirlinkDR — Hub Session Tables
================================================================

TABLE: Hub_Sessions
  SessionID     uniqueidentifier PK
  EmployeeCode  varchar(20)
  AgentType     varchar(20)  -- 'CLAUDE' | 'DINO' | 'LX'
  StartedAt     datetime
  LastActivity  datetime

TABLE: Hub_Messages
  MessageID   int PK IDENTITY
  SessionID   uniqueidentifier FK → Hub_Sessions
  Role        varchar(20)  -- 'user' | 'claude' | 'dino'
  Content     nvarchar(MAX)
  CreatedAt   datetime

================================================================
## STORED PROCEDURES — DB: AirlinkDR
================================================================

SP_Hub_NewSession
  @EmployeeCode varchar(20)
  @AgentType    varchar(20)
  Returns: SessionID (uniqueidentifier as varchar)

SP_Hub_SaveMessage
  @SessionID  uniqueidentifier
  @Role       varchar(20)
  @Content    nvarchar(MAX)
  Returns: MessageID

SP_Hub_LoadHistory
  @SessionID  uniqueidentifier
  @Limit      int = 40
  Returns: MessageID, Role, Content, CreatedAt (ASC order)

## NOTES
================================================================
- Cedula format in DB: XXX-XXXXXXX-X (with hyphens)
- LastFour validation: RIGHT(REPLACE(Cedula,'-',''), 4)
- Employee code format: DR + 4 digits zero-padded (DR0002)
- Photo sync: \\ADRDB1\Fotos Empleados\{Numero}.jpg → OneDrive\EmployeePics\{Codigo}.jpg
- Photo naming: file number (2, 20, 200) → DR code (DR0002, DR0020, DR0200)
